# הבית הצהוב — מקור הידע המרכזי של המערכת

> **מסמך זה הוא מקור האמת היחיד של הפרויקט.**
> מבוסס על שיחות האפיון והפיתוח המלאות.
> יש לקרוא אותו בתחילת כל סשן עבודה לפני כל פעולה.
> יש לעדכנו בכל החלטה מהותית חדשה.

## 1. מהות המיזם

**הבית הצהוב (Habayit Hatsahov)** — אינדקס עסקים והטבות של קהילת אוהדי מכבי תל אביב, מאוהדים לאוהדים.

- **ללא מטרות רווח בשלב זה**
- עסקים בבעלות אוהדים מפרסמים את עצמם
- אוהדים מאושרים נהנים ממגוון עסקים ומהנחות בלעדיות
- האתר **סגור לחלוטין לאנשים שאינם רשומים ומאושרים** — מלבד דפי ההרשמה

## 4. סוגי משתמשים והרשאות

### 4.1 אורח (Guest)
- גישה ל-`fan-register.html` — הרשמת אוהד
- גישה ל-`business.html` — הרשמת עסק (**פתוח לכולם, ללא צורך להיות חבר מאושר קודם**)
  - הטופס כולל בתוכו אימות אוהד מובנה

### 4.2 חבר מאושר (Approved Fan Member)
- גישה ל-`home.html` — אינדקס עסקים
- גישה ל-`business-page.html` — עמוד עסק בודד
- גישה ל-`profile.html` — פרופיל אישי הכולל תעודת חבר (**חשוף לחבר מאושר**)
- גישה ל-`card.html` — תעודת חבר standalone
- חשיפת קופונים ומימוש הטבות

### 4.3 מנהל (Admin)
- גישה ל-`admin.html` — פאנל ניהול
- אישור/דחיית בקשות הצטרפות
- **⚠️ אין כרגע מנגנון אימות — admin.html נגיש לכולם. יש לאבטח עם Firebase Auth.**

## 5. מבנה הדפים

```
index.html           ← redirect → business.html (meta refresh)
fan-register.html    ← דף נחיתה + הרשמת אוהד (אורחים)
business.html        ← הרשמת עסק לאינדקס (אורחים + חברים, כולל אימות אוהד)
firebase-config.js   ← הגדרות Firebase משותפות (מיובא ע"י business.html + admin-businesses.html)
home.html            ← אינדקס עסקים ראשי — חברים מאושרים
business-page.html   ← עמוד עסק מלא (?id=N) — לשיתוף לינקים
profile.html         ← פרופיל אישי + תעודת חבר + הגדרות — חברים מאושרים
card.html            ← תעודת חבר standalone
admin.html           ← פאנל ניהול
nav.html             ← ⚠️ DEV ONLY — למחיקה לפני העלאה
tasks.html           ← ⚠️ DEV ONLY — למחיקה לפני העלאה
```

### זרימת ניווט
```
[אורח]
index.html ──(מילוי טופס)──→ ממתין לאישור מנהל
business.html ──(מילוי טופס)──→ ממתין לאישור מנהל

[מנהל]
admin.html ──(אישור)──→ נשלח WhatsApp + מייל לאוהד

[חבר מאושר]
home.html
  ├── לחיצה על עסק → bottom sheet קופץ (חוויה ראשית)
  ├── "פתח דף מלא" → business-page.html?id=N (לשיתוף)
  └── לחיצה על פרופיל → profile.html → card.html
```

## 6. אפיון כל דף

### 6.1 דף נחיתה והרשמת אוהד (`index.html`)

**Hero:**
- לוגו מכבי (SVG רשמי, צבוע צהוב) עם אנימציית float
- כותרת "הבית הצהוב"
- Tagline + כפתור CTA + Stats strip

**4 Feature cards:** מחזקים עסקים / ספקים מהימנים / הנחות / משפחה

**טופס הרשמה — שדות:**
- שם פרטי, שם משפחה, טלפון, אימייל
- **אימות אוהד — 2 נתיבים:**
  - מנוי פעיל → בחירת שער (1/2/7/8/11/13) + מספר מנוי
  - לא מנוי → "האם היית מנוי?" + תמונה מהאצטדיון + קישור פייסבוק
- שאלה "בעל עסק?" → **לא חובה, מופיעה בתחתית הטופס** → מי שענה "כן" מקבל למייל לינק להרשמת עסק לאחר אישור חברות
- **הוספת בני משפחה (עד 2):** שם, קרבה, טלפון, אימייל — אופציונלי
- תנאי שימוש + כפתור שליחה

**כללי בני משפחה (מאפיין שהוחלט):**
- עד 2 בני משפחה תחת חשבון אחד
- אישור המשתמש הראשי = אישור אוטומטי לבני המשפחה
- כל בן משפחה מקבל תעודת חבר נפרדת

### 6.2 פאנל ניהול (`admin.html`)

**Stats row:** סה"כ נרשמים / ממתינים / מאושרים

**רשימת בקשות (expand/collapse per כרטיס):**
- Avatar (אותיות ראשונות), שם, אימייל, status badge
- **זמן המתנה (מאפיין שהוחלט):**
  - 🟢 עד 6 שעות — "fresh"
  - 🟠 עד 24 שעות — "medium"
  - 🔴 מעל יום — "long"
- **תאריך אישור (מאפיין שהוחלט):** כשמאשרים — מוצג "✓ אושר ב-[תאריך ושעה]"
- Detail grid: טלפון, אימייל, שער, מנוי, עסק?
- כפתורי אישור / דחייה
- כפתור WhatsApp (פותח עם הודעה מוכנה — ידני)
- כפתור מייל (פותח לקוח מייל — ידני)

**EmailJS (לא מחובר עדיין — משימה לעתיד):**
- בעת אישור — מייל נשלח אוטומטי ללא פעולה ידנית

### 6.3 תעודת חבר דיגיטלית (`card.html`, מוטמע גם ב-`profile.html`)

**מתי נוצרת:** רק לאחר אישור מנהל (לא בהרשמה)

**מה מוצג על התעודה (מאפיין שהוחלט):**
- לוגו "הבית הצהוב" + לוגו מכבי
- שם מלא
- מספר חבר (ממוספר אוטומטי, `#XXXXX`)
- תאריך הצטרפות
- שדה "שער" — רק אם הוא מנוי פעיל
- "חבר מאושר ✓"
- **אין:** תמונה, תוקף

**סגנון:** כרטיס אשראי שוכב, aspect-ratio 1.586, **רקע צהוב דומיננטי (גרדיאנט #FFDE00→#E6C200), פרטים וטקסט בכחול נייבי (#002a75) לניגודיות, פס ושבב בכחול**

**איך האוהד מקבל ומשתמש:**
- נשלח כלינק אישי בווטסאפ ובמייל לאחר אישור
- נגיש באזור האישי (`profile.html`)
- כפתור "שמור לגלריה" (html2canvas scale=3)
- מציג בחנות פיזית מהמסך / מהגלריה

**בני משפחה:** תעודה נפרדת לכל אחד, כולל "בן/בת משפחה של [שם]"

### 6.4 אינדקס עסקים (`home.html`) — Dark Theme

**עיצוב כהה (אושר ומומש):**
- רקע: `#151C2E`, כרטיסים: `#243362`, nav: `#0f1524`
- צהוב ראשי: `#FFDE00`
- Skeleton cards מיידיים בטעינה (ללא loading overlay)

**Top nav (sticky):**
- לוגו + כפתור "+ הוסף עסק" (צהוב)
- כפתור "← תפריט" (DEV ONLY)

**Bottom nav (קבוע בתחתית — כל הדפים):**
- 4 פריטים: 🏠 בית / 🔍 חיפוש / 📍 קרוב אליך / 👤 פרופיל
- לחיצה על "חיפוש" → גלילה לראש הדף + פוקוס שדה חיפוש
- לחיצה על "קרוב אליך" → בקשת הרשאה / הפעלת מיקום + גלילה לסקציה

**Category pills:**
- `flex-wrap` — כל הקטגוריות גלויות ב-2 שורות, ללא גלילה אופקית
- 10 קטגוריות (ללא emoji): אוכל ושתייה / בעלי מקצוע / ביגוד ואופנה / בריאות ויופי / פיננסים / טכנולוגיה / משפטים / ספורט ופנאי / אחר / הכל

**סקציות (scroll אופקי per סקציה):**
1. ✨ **חדש אצלנו** — לפי `isNew` + `newUntil`
2. 💛 **הטבות שוות** — עסקים עם `discount`
3. 🔥 **הכי פופולריים** — Top 5 לפי `clicks`
4. 🛒 **קניה אונליין** — עסקים עם `online: true`
5. 📍 **קרוב אליך** — בקשת GPS, Euclidean distance. ללא מיקום: הודעה רכה "אפשר מיקום לראות עסקים קרובים". עם מיקום: tag עם שם העיר (Nominatim reverse geocoding) + כפתור "שנה"

**כרטיס עסק (Business Card):**
- לוגו (תמונה / אותיות ראשונות על גרדיאנט קטגוריה)
- שם העסק
- Tag: "בעל מקצוע" (לבעלי מקצוע) / "פיזי + אונליין" / "אונליין"
- מקצוע ספציפי כטקסט מתחת ל-tag (לבעל מקצוע) / שם קטגוריה לשאר
- ההנחה מוצגת ישירות (ללא לחיצה נוספת)
- **אין שורת עיר על הכרטיס**

**Analytics (מאפיין שהוחלט):**
- כל לחיצה על עסק = click נרשם
- **הקליק עצמו הוא האנליטיקס** — אין צורך בכפתור "חשוף" נפרד
- נשמר ב-localStorage (`_hb_clicks_{id}`) עד Firebase

**חשיפת הטבה:**
- **אונליין** → קוד קופון מוצג ישירות (ללא לחיצת חשיפה — מוחלט)
- **פיזי** → הוראה להציג תעודת חבר

**חוויה ראשית במובייל: Bottom Sheet (מוחלט)**

- לא יוצאים מהאינדקס

**תגיות בראש הגיליה (sheetMeta):**
- קטגוריה ראשית
- מקצוע ספציפי (לבעל מקצוע, אם הוגדר)
- 🌐 אונליין (אם רלוונטי)
- 📍 עיר (רק אם קיים)

**בדסקטופ:**

**`business-page.html` — נשמר לשיתוף לינקים בלבד**
- URL נפרד לכל עסק (`?id=N`)

### 6.6 הרשמת עסקים (`business.html`)

**פתוח לכולם — אורחים וחברים. כולל אימות אוהד מובנה.**

**Stepper 4 שלבים:**

**שלב 1 — בעל העסק:**
- הערה חמה "אפשר לרשום גם עסק של בן משפחה"
- שם, טלפון, אימייל
- **שדה קרבה** לבעל המנוי (אופציונלי) — בן/בת זוג, הורה, אח/ות, ילד/ה, אחר. ברירת מחדל: "העסק שלי — לא רלוונטי"
- אימות אוהד (זהה ל-index.html)

**שלב 2 — פרטי העסק:**
- סוג עסק (פיזי/אונליין/מקצוע, עד 2 בחירות)
- **בחירת "בעל מקצוע" מסמנת אוטומטית קטגוריה ראשית "בעלי מקצוע" בשלב 3** (ביטול הבחירה מוריד את הסימון)
- שם, תיאור
- **סניפים** (לפיזי) — עד 5 סניפים, כל אחד עם עיר + כתובת. כפתור "+ הוסף סניף" עם מחיקה ומספור אוטומטי
- אזורי פעילות (לבעל מקצוע), אתר חנות (לאונליין)
- טלפון, WhatsApp, מייל שירות

**שלב 3 — קטגוריות והטבות:**
- קטגוריה — 9 pill grid (ראשית + עד 2 משניות)
- הטבה לחברים — toggle (דיפולט=כן) + פרטי הטבה + **שורת רעיונות להטבות** (אחוז הנחה, הנחה על קטגוריה, תוספת חינם, קנה X קבל Y, ייעוץ חינם, משלוח חינם)
- **סוג מימוש: chips — ניתן לבחור גם פיזי וגם אונליין** (כפתורי chip, לא select)
- קוד קופון (אם אונליין נבחר)
- שעות פעילות, נוכחות דיגיטלית (אתר, אינסטגרם, פייסבוק)
- **הערה חופשית** — "רוצה לכתוב לנו משהו?"

**שלב 4 — סיכום ושליחה:**
- תצוגת כל הפרטים שמולאו (בעל העסק, פרטי העסק, קטגוריות והטבות, הערה)
- אפשרות חזרה לעריכה בכל שלב
- כפתור "שלח בקשת הצטרפות לאינדקס"

## 9. חוקים עסקיים

1. **רק אוהדי מכבי** — האינדקס סגור לציבור.
2. **אימות אוהד בשני נתיבים** (קיים ב-index.html וב-business.html):
   - מנוי פעיל → שער + מספר מנוי
   - לא מנוי → תמונה מאצטדיון + קישור פייסבוק
3. **בעל עסק יכול להירשם ישירות** ב-business.html ללא צורך להיות חבר קודם.
4. **עד 2 בני משפחה** — אישור ראשי = אישור אוטומטי לבני משפחה.
5. **תעודת חבר נוצרת רק לאחר אישור מנהל.**
6. **תעודה נפרדת לכל בן משפחה**, כולל ציון "בן/בת משפחה של [שם]".
7. **ספירת clicks = analytics** — הלחיצה עצמה מספיקה, אין "כפתור חשיפה" מיותר.
8. **קוד קופון מוצג ישירות** בלחיצה על עסק (לא דורש לחיצה נוספת לחשיפה).
9. **עסק "חדש"** = `isNew: true` AND `new Date() < newUntil` — משך מוגדר ע"י מנהל.
10. **"פופולריים"** = Top 5 ממוין לפי `clicks`.
11. **תפריט** — פיצ'ר ייחודי לקטגוריה `food` בלבד.
12. **אפשר לרשום עסק של בן משפחה** — שדה קרבה לבעל המנוי (אופציונלי).
13. **עסק עם כמה סניפים** — עד 5 סניפים, כל אחד עם עיר + כתובת.
14. **סוג מימוש הטבה** — ניתן לבחור גם חנות פיזית וגם אונליין (chips, לא select בודד).
15. **ולידציה חוסמת** — לא ניתן לדלג על שלב בטופס הרשמת עסק בלי למלא שדות חובה.
16. **בחירת "בעל מקצוע"** בשלב 2 מסמנת אוטומטית קטגוריה ראשית "בעלי מקצוע" בשלב 3 (ביטול הבחירה מסיר את הסימון).
17. **אישור עסק ← מאשר אוטומטית את חברות האוהד המקושרת** — כשמנהל מאשר עסק (יחיד או bulk), המערכת מחפשת ב-Firestore חבר עם אותו אימייל (`ownerEmail`/`email`) ומעדכנת אותו ל-`status: 'approved'` (+ `familyApproved: true`, ביטול `revoked` לכל בני המשפחה) — אלא אם הוא כבר מאושר.
18. **סרגל ניווט תחתון קבוע עם 5 פריטים לבעלי עסקים** — בכל דף עם bottom-nav (home, profile, card, business-dashboard) מוצגים 4 הפריטים הרגילים (בית/חיפוש/קרוב אליך/פרופיל), ואם המשתמש המחובר הוא בעל עסק (`isBusinessOwner: true`) מתווסף פריט חמישי "🏪 העסק שלי" שמוביל ל-`business-dashboard.html?token=X` (הטוקן נשלף מ-Firestore לפי `ownerPhone`/`ownerEmail`). ניווט פנימי בין תצוגות (כמו "העסק שלי" מול "תנאי שימוש") הוא היוצא מן הכלל היחיד שעדיין נשאר כניווט פנימי, לא בסרגל התחתון.
19. **קישור "פרופיל" מתוך דשבורד עסק תמיד מציג את החבר המקושר לעסק הספציפי** — business-dashboard.html מעביר ל-profile.html את הטוקן של העסק כ-`?bizToken=X`, ו-profile.html (כשקיים הפרמטר) מזהה לפיו את בעל העסק (accessToken → ownerEmail → member לפי אימייל) במקום להסתמך על session כללי. מונע מצב שבו "פרופיל" מדשבורד עסק א' מציג בטעות את המשתמש שמחובר במכשיר, ולא את בעל העסק עצמו.
20. **"זיכרון" עסק אחרון שנצפה (dev בלבד, ללא Auth אמיתי)** — כל עוד ה-Auth guard מושבת, אין למערכת דרך לדעת "מי אתה" בלי session אמיתי. לכן: כשנפתח business-dashboard.html דרך טוקן ישיר (`?token=X`), הטוקן נשמר ב-`sessionStorage.hb_lastBizToken`. כפתור "🏪 העסק שלי" בסרגל התחתון (home/profile/card) ובכרטיס "העסק שלי" בפרופיל — כשאין session אמיתי — משתמשים בטוקן השמור הזה, ורק אם אין כזה נופלים לעסק ברירת מחדל לבדיקות ("יהב מטבחים"). כך ניווט חופשי בין דפים (בית ↔ עסק ↔ פרופיל) נשאר עקבי לאותו עסק שנבדק, גם ללא login אמיתי. **בפרודקשן, לאחר הפעלת Auth guard, זה יתבסס אך ורק על session אמיתי (`hb_memberId`) ולא יהיה צורך במנגנון הזה.**

## 16. מצב נוכחי

**שלב: Auth + אינדקס מחוברים — הפלטפורמה עובדת end-to-end**

- כל הדפים בנויים — UX/UI מוכנים
- **business.html מחובר ל-Firebase Firestore** — הגשות נשמרות ב-collection `businesses`
- **fan-register.html מחובר ל-Firebase Firestore** — הגשות נשמרות ב-collection `members` עם `status: 'pending'`
- **admin-businesses.html קורא מ-Firestore בזמן אמת** (onSnapshot)
- **ולידציה חוסמת** — לא ניתן לדלג על שלבים ב-business.html בלי למלא שדות חובה
- **אירוח ב-GitHub Pages** — `habayit-hatsahov.github.io/mtabusiness/`
- **index.html עושה redirect ל-business.html** (כדי שהכתובת הראשית תוביל להרשמת עסקים)
- **fan-register.html** — דף הרשמת אוהד (שונה שם מ-index.html)
- דשבורד ניהול פוצל ל-3 קבצים: admin.html (hub), admin-members.html, admin-businesses.html
- שאר הדפים (home, profile, card) עדיין עם נתונים hardcoded
- **business.html** — בהרשמה נוצר `accessToken` (UUID) + שמירה כפולה: `businesses` + `members` (isBusinessOwner: true)
- **business-dashboard.html** — דשבורד בעל עסק דרך token, תנאי שימוש בכניסה ראשונה, עריכה עם pendingChanges
- **admin-businesses.html** — badge שינויים ממתינים, diff + אשר/דחה, לינק דשבורד במודל האישור
- **Firebase Phone Auth** — `login.html` עם OTP, guard על home/profile/card/admin-*
- **`home.html`** מקבל עסקים אמיתיים מ-Firestore (status=approved)
- **`profile.html`** ו-**`card.html`** טוענים נתוני חבר אמיתיים מ-Firestore
- **admin guard** — בודק `isAdmin: true` על הדוק ב-members
- **Firebase Storage** — מחובר ב-admin-businesses (לוגו + גלריה), admin-members (הוכחה), business.html (הוכחת אוהד)
- **קטגוריות sport/other** — נוספו ל-home.html (גריד + לייבלים + צבעים)
- **card.html** — עבר Dark theme מלא, top-nav + bottom-nav מאוחדים עם home/profile, תוקן dev-fallback שהיה שבור (הפנה תמיד ל-login.html)
- **עיצוב תעודה חדש (רקע צהוב דומיננטי + טקסט כחול נייבי)** — סונכרן ב-card.html, profile.html, business-dashboard.html
- **business-dashboard.html** — עבר Dark theme מלא (topbar/biz-hero/profile-hero אוחדו ל-`var(--nav-bg)` ללא הפרדת רקע כחול, כרטיסים/מודלים/טפסים בפלטה הכהה). **הניווט בין "העסק שלי" ל"הפרופיל שלי" עבר מטאבים בטופבר לסרגל תחתון קבוע** (bottom-nav תואם לשאר האתר) — הטופבר נשאר עם הלוגו בלבד
- **admin-businesses.html** — אישור עסק (יחיד או bulk) מאשר כעת אוטומטית גם את חברות האוהד המקושרת (חיפוש לפי אימייל, כולל אישור בני משפחה)
- **סרגל תחתון קבוע עם 5 פריטים לבעלי עסקים** — נוסף פריט "🏪 העסק שלי" (מוצג רק לבעלי עסקים) ב-home.html, profile.html, card.html, business-dashboard.html — מוביל ל-business-dashboard.html?token=X
- **business-dashboard.html** — הוסר הטאב הפנימי "הפרופיל שלי" (כפול לתוכן ב-profile.html); הדף מכיל רק את "העסק שלי" + "תנאי שימוש"; הניווט לדפים אחרים עבר לסרגל התחתון האחיד
- **תוקן באג בכפתור "העסק שלי" (מצב dev ללא session אמיתי)** — הטוקן הפיקטיבי `token=demo` לא הצביע על עסק אמיתי ב-Firestore וגרם ל"קישור לא תקין". הוחלף בטוקן אמיתי לבדיקות (`c6342c93-a350-42b8-ad87-95fa005c8983` — עסק "יהב מטבחים") ב-home.html, profile.html, card.html
- **תוקן באג: "פרופיל" מדשבורד עסק הציג פרופיל לא נכון** — לחיצה על "פרופיל" מתוך business-dashboard.html הסתמכה רק על session הנוכחי (`hb_memberId`), ולכן הציגה לעיתים פרופיל לא קשור לבעל העסק שנצפה (או את ה-dev fallback הכללי). **תוקן:** business-dashboard.html מעביר עכשיו `profile.html?bizToken=X` (אותו טוקן עסק), ו-profile.html כשמזהה `bizToken` ב-URL מחפש את העסק לפי `accessToken`, מוצא את האימייל של הבעלים, ומאתר את החבר המקושר לפי אימייל — כך הפרופיל תמיד תואם לבעל העסק הנצפה, לא ל-session
- **⚠️ Auth guard מושבת זמנית** על כל הדפים — להפעיל לפני העלאה לפרודקשן
- **סבב עיצוב Dark theme הושלם על כל הדפים** — admin.html, admin-members.html ו-admin-businesses.html עברו את ההמרה האחרונה: modal אישור, view-sheet, modal עריכה (כולל stepper 5 שלבים, טפסים, pill-grid קטגוריות, אזור העלאת מדיה) ובחירה מרובה (bulk select) — כולם תואמים כעת לפלטת הכהה של שאר האתר. **משימה 4 בתוכנית העבודה (סעיף 14) הושלמה במלואה**
- **הצעד הבא:** EmailJS — צריך ליצור חשבון + 2 תבניות (member_approved, business_approved) ואז לחבר בקוד
